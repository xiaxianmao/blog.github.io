类型

常量 Constants

  • 不是变量，编译时被创建，只能是数字 字符 字符串 或 bool
  • 定义的表达式必须能被编译器识别的常量表达式。如 << >>
  • 枚举常量 iota 默认0 自增+1

变量 Variables

  • 初始值可以是调用函数获取的值

Init函数 The init function

  • 每个文件可以拥有多个init函数，调用后意味着初始化结束
  • 导入包 > 变量声明 > init

数组 Array

  • 数组是值。将一个数组赋予另一个数组会复制其所有元素。 利用指针传递，避免重复开销
  • 特别地，若将某个数组传入某个函数，它将接收到该数组的一份副本而非指针。
  • 数组的大小是其类型的一部分。类型 [10]int 和 [20]int 是不同的

一般并不常用array,一般使用slice切片

切片 Slices

传递了底层数组的指针,切片拷贝引用的是同一个数组
不定长度，当len>cap 时，cap翻倍，>1000时 每次增加0.25

二维切片 Two-dimensional slices

数组的数组  切片的切片  每个切片（数组）的长度可以不同。可以分配初始化长度和容量

映射 Map

引用类型传递
获取不存在的key，返回的是该类型的0值
区分是否存在需要使用多重赋值 如 if seconds, ok := timeZone[tz]...
删除key  delete(timeZone,'key')

接口 interface

信道 channel

make 来分配内存，提供缓冲区大小，0表示不带缓冲/同步的信道

用来处理多协程中的数据交换（通过通信共享内存）

带缓冲的信道可被用作信号量，例如限制吞吐量。在此例中，进入的请求会被传递给 handle，它从信道中接收值，处理请求后将值发回该信道中，以便让该 “信号量” 准备迎接下 一次请求。信道缓冲区的容量决定了同时调用 process 的数量上限。 //限流 熔断

这种特性通常被用 来实现安全、并行的多路分解。

信道中的信道 Channels of channels

声明&初始化

  • new分配：内存置零。它返回一个指针，指针指向新分配的零值。表达式 new(File) 和 &File{} 是等价的。适用于任何类型的创建
  • make分配：只用于初始化切片、映射和信道，且不返回指针

类型断言

错误 Errors

多值返回。通常附带错误信息 error类型
信息一般包含 文件名、操作和触发的操作系统错误
Don’t Panic（不要抛出panic）

程序发生意外
应该尽量避免panic

Recovery

特性

并发 Concurrency

通过通信共享内存 Share by communicating

Go 语言另辟蹊径，它将共享的值通过信道传递，实际上，多个独立执行的线程从不会主动共享。 在任意给定的时间点，只有一个 goroutine 能够访问该值。数据竞争从设计上就被杜绝了

Do not communicate by sharing memory; instead, share memory by communicating.
不要通过共享内存来通信，而应通过通信来共享内存。

  • 引用计数通过为整数变量添加互斥锁来很好地实现
  • 但作为一种高级方法，通过信道来控制访问能够让你写出更简洁，正确的程序

goroutine

  • Goroutine 的设计隐藏了线程创建和管理的诸多复杂性

  • go doSomething()当调用完成后，该goroutine 也会安静地退出

  • go func(){}() //闭包

并行化 Parallelization

这些设计的另一个应用是在多 CPU 核心上实现并行计算。如果计算过程能够被分为几块可独 立执行的过程，它就可以在每块计算结束时向信道发送信号，从而实现并行处理。

runtime.GOMAXPROCS(NCPU) or GOMAXPROCS 设置多核心执行

Go 仍然是种并发而非并行的语言，且 Go 的模型并不适合所有的并行问题

并发和并行的区别 并发：同时处理的任务书 并行：在多个核处理任务。 一条马路多车道 vs 多条马路
